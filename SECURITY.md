# Security Policy

We take security issues seriously. If you discover a vulnerability in an InPrime repository or service, report it privately to `tech@inprime.in`.

## How To Report

- Email `tech@inprime.in` with the subject line `[Security] <repository or service name>`.
- Include the affected repository, the impact, reproduction steps, and any proof-of-concept or remediation notes you have.
- Do not open a public GitHub issue or pull request for the initial report.

## What To Expect

- We aim to acknowledge new reports within 2 business days.
- We will validate the issue, coordinate remediation, and work with you on disclosure timing when needed.

## Supported Versions

- Security fixes are applied to actively maintained default branches and currently supported production services.
